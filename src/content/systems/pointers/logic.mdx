## What pointers solve

High-level languages hide memory addresses behind references or objects, but systems programming needs direct access to memory. Pointers let a program store and manipulate the address of a variable, enabling dynamic data structures, array traversal, and call-by-reference function parameters.

## Key operations

1. **Address-of (`&`)**: obtains the memory address of a variable.
2. **Dereference (`*`)**: accesses the value stored at the address a pointer holds.
3. **Pointer arithmetic**: adding `n` to a pointer advances it by `n × sizeof(T)` bytes.
4. **Null pointer**: a pointer that points to nothing (address `0`); dereferencing it crashes.

## Pseudocode

~~~c
int main(void) {
    int  x  = 42;      /* x is an int */
    int *p  = &x;      /* p holds the address of x */
    *p = 99;           /* dereference p — x is now 99 */

    int  y  = *p;      /* read via dereference — y is 99 */

    int **pp = &p;     /* pointer to pointer */
    **pp = 7;          /* double dereference — x is now 7 */

    return 0;
}
~~~

## Common pitfalls

- **Null dereference**: accessing `*NULL` causes a segmentation fault.
- **Dangling pointer**: using a pointer after the target memory has been freed.
- **Type mismatch**: assigning an `int*` to a `float*` and dereferencing violates strict aliasing.
- **Off-by-one in pointer arithmetic**: walking past the end of an array.
- **Forgetting `&` vs `*`**: `p = &x` sets the pointer; `*p = v` sets the target.
