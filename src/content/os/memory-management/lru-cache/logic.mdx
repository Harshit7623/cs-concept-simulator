## What LRU caching solves

LRU (Least Recently Used) caching stores a limited number of items and automatically evicts the item that was accessed least recently when new items need space. It exploits temporal locality — recently accessed data is likely to be accessed again.

## Algorithm steps

1. Look up the key in the hash map.
2. If found (cache hit):
   - Move the corresponding node to the head of the linked list.
   - Return the value.
3. If not found (cache miss):
   - If the cache is full, remove the node at the tail (LRU) and delete its key from the hash map.
   - Create a new node, insert it at the head of the linked list.
   - Add the key to the hash map.

## Pseudocode

~~~c
typedef struct Node {
    int key;
    int value;
    struct Node *prev;
    struct Node *next;
} Node;

typedef struct {
    Node *head;
    Node *tail;
    Node *map[10000];
    int capacity;
    int size;
} LRUCache;

int lruGet(LRUCache *cache, int key) {
    if (cache->map[key] == NULL) {
        return -1;
    }
    Node *node = cache->map[key];
    moveToHead(cache, node);
    return node->value;
}

void lruPut(LRUCache *cache, int key, int value) {
    if (cache->map[key] != NULL) {
        Node *node = cache->map[key];
        node->value = value;
        moveToHead(cache, node);
        return;
    }
    if (cache->size == cache->capacity) {
        removeTail(cache);
    }
    Node *node = createNode(key, value);
    addToHead(cache, node);
    cache->map[key] = node;
    cache->size++;
}
~~~

## Common pitfalls

- Forgetting to update the hash map when a node is evicted from the list.
- Not handling the case where the cache has only one node (head == tail).
- Breaking the doubly-linked list pointers when moving or removing nodes.
- Using O(n) operations in get/put defeats the purpose of the cache.
