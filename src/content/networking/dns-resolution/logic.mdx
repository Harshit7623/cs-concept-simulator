## What DNS resolution solves

DNS translates human-readable domain names (like example.com) into machine-routable IP addresses. Without DNS, users would need to memorize numeric IP addresses for every website.

## Steps

1. Client queries the recursive resolver for example.com.
2. Recursive resolver queries a root nameserver.
3. Root responds with a referral to the .com TLD nameserver.
4. Recursive resolver queries the .com TLD nameserver.
5. TLD responds with a referral to the authoritative nameserver for example.com.
6. Recursive resolver queries the authoritative nameserver.
7. Authoritative responds with the IP address.
8. Recursive resolver returns the IP to the client and caches the result.

## Pseudocode

~~~c
typedef struct {
    char domain[256];
    char ip[16];
} DNSRecord;

typedef struct {
    DNSRecord cache[100];
    int cacheSize;
} Resolver;

char *recursiveLookup(Resolver *res, char *domain) {
    for (int i = 0; i < res->cacheSize; i++) {
        if (strcmp(res->cache[i].domain, domain) == 0) {
            return res->cache[i].ip;
        }
    }
    char *rootResult = queryNameserver("root", domain);
    if (isAuthoritative(rootResult)) {
        return cacheResult(res, domain, rootResult);
    }
    char *tld = extractTLD(domain);
    char *tldResult = queryNameserver(tld, domain);
    if (isAuthoritative(tldResult)) {
        return cacheResult(res, domain, tldResult);
    }
    char *authResult = queryNameserver(domain, domain);
    return cacheResult(res, domain, authResult);
}
~~~

## Common pitfalls

- Cache poisoning — an attacker injects a fake DNS record into a resolver's cache.
- Assuming DNS resolution always succeeds — networks may have blocked or misconfigured resolvers.
- Ignoring TTL values — stale cached records can cause connectivity issues after IP changes.
- DNS over TCP vs UDP — large responses require TCP fallback when the truncation bit is set.
