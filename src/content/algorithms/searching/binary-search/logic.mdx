## What binary search solves

Binary search finds the position of a target value in a sorted array in logarithmic time. Instead of scanning every element, it eliminates half the remaining range with each comparison.

  <section className="logic-card"><span className="logic-eyebrow">Invariant</span><h3>Search range always contains the answer</h3><p>The target — if present — must lie between <code>low</code> and <code>high</code>. Each iteration shrinks this interval without skipping the target.</p></section>

## Algorithm steps

1. Initialize low = 0, high = n - 1.
2. While low ≤ high:
   - Compute mid = low + (high - low) / 2.
   - If arr[mid] equals target, return mid.
   - If target < arr[mid], set high = mid - 1.
   - If target > arr[mid], set low = mid + 1.
3. If the loop exits, the target is not in the array.

## Pseudocode

~~~c
int binarySearch(int arr[], int n, int target) {
    int low = 0;
    int high = n - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;
        if (arr[mid] == target) {
            return mid;
        }
        if (target < arr[mid]) {
            high = mid - 1;
        } else {
            low = mid + 1;
        }
    }
    return -1;
}
~~~

## Common pitfalls

- Forgetting the array must be sorted — binary search on unsorted data gives garbage.
- Integer overflow when computing `(low + high) / 2` on very large arrays; use `low + (high - low) / 2`.
- Off-by-one in the loop condition (`low < high` vs `low <= high`) can miss the last element.
- Not handling the case where the target appears multiple times (lower/upper bound variants).
